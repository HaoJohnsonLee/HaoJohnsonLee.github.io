ThreadPoolExecutor

Posted on 2018-07-22
Words count in article 1,507 | Reading time 7 mins

Root Constructor

初始化各个线程池参数:

  1. corePoolSize : 当allowCoreThreadTimeOut为默认值时，线程池中最小线程数为corePoolSize,否则线程池中最小线程数为 0
  2. maximumPoolSize : 线程池中最大线程数
  3. keepAliveTime & unit : 当一个线程在 keepAliveTime时间段内空闲，且当前线程数大于当前系统容忍的最小线程数，该线程会被回收。
  4. workQueue : 阻塞的工作线程队列，当任务量大于线程总数时，无法获得活动线程的任务将暂存在该队列中。
  5. threadFactory : 生产线程的工厂，该接口只有一个方法Thread newThread(Runnable r);，默认实现为DefaultThreadFactory,返回一个优先级为Thread.NORM_PRIORITY的非守护线程。
  6. handler : 当任务数大于阻塞队列与最大线程数的和时，将会调用方法void rejectedExecution(Runnable r, ThreadPoolExecutor executor);,默认实现为AbortPolicy,抛出一个RejectedExecutionException异常。

如何合理设置参数值

  • tasks，每秒需要处理的最大任务数量
  • tasktime，处理第个任务所需要的时间
  • responsetime，系统允许任务最大的响应时间
  1. corePoolSize = tasks * tasktime (具体遵守8020原则)
  2. queueCapacity = tasks * responsetime (该值不应设置为默认值，否则会导致系统响应过慢，线程池线程得不到增长)
  3. 具体情况还应具体分析，公式只是参考，要考虑到CPU、内存等实际情况分配资源。

一些状态属性

属性ctl实际上表示了两个属性，其高 3 位表示运行状态，低 29 位表示当前worker总数。
一共定义了五种线程池状态:

  1. RUNNING : 能够接收新任务并且处理队列中任务
  2. SHUTDOWN : 不接收新任务但能够处理队列中任务
  3. STOP : 不接收新任务也不处理队列中任务，并且将正在处理的任务中断
  4. TIDYING : 所有的线程都已经结束，workerCount为 0， 线程转换到TIDTING状态将会运行terminated() hook method
  5. TERMINATED : terminated() 方法运行完成

    状态转换

  6. RUNNING -> SHUTDOWN : 调用shutdown()方法
  7. (RUNNING or SHUTDOWN) -> STOP : 调用 shutdownNow()方法
  8. SHUTDOWN -> TIDYING : 队列与线程池任务为空
  9. STOP -> TIDYING : 线程池任务为空
  10. TIDYING -> TERMINATED : terminated() 方法运行完成
